duckPond

Team

Anna Lamontagne, Abdul aziz Hamoui

My Roles

Visual Designer, UI Designer, Front-End Developer

Duration

Jan 27th – March 3rd 2025

Overview

Duck Pond is a responsive website created for a fictional yoga studio company. The goal was to design and build a clear, user-friendly experience that could support browsing, purchasing, and general information in a way that felt cohesive and easy to move through.

Working in a team of two, we developed the site from scratch using HTML and CSS, focusing on usability, responsiveness, and a consistent visual identity.

Background

The project began with defining the brand itself. I initially developed the visual identity, including typography, colour, buttons, and other interface assets, before moving into the design and development of the site with my teammate.

Without using frameworks, every part of the interface had to be built manually, which made structure and planning especially important. The project became a useful exercise in thinking about how brand, layout, and code all connect.

Process

We started by defining the company’s structure, products, and user flow. I created early wireframes to map out the layout and ensure the experience felt clear before moving into development.

From there, I translated the visual identity into a system of reusable components, including buttons, layouts, and interface elements that could scale across the site.

Development was done using HTML and CSS, with a strong focus on responsiveness. Rather than treating responsiveness as an afterthought, we designed for multiple screen types from the beginning, including less conventional formats like smaller devices and large displays.

This required a shift in how I thought about layout. Instead of designing for a single screen, I had to think in systems that could adapt across contexts.

Solution

The final site includes product listings, a purchase flow, and supporting content, all structured within a responsive system. Reusable components helped maintain consistency across pages, while the visual identity created a cohesive and recognizable experience.

The result was a front-end system that balanced branding, usability, and structure in a way that felt polished without relying on external frameworks.

Results

The result was a fully functional front-end website that performs consistently across devices and screen sizes. The project demonstrated how thoughtful structure and component-based design can support both usability and scalability.

It also gave me a much better understanding of how visual design decisions hold up once they are translated into code.

Learnings

This project changed how I think about responsiveness. Instead of designing for a fixed layout, I began to see design as something that needs to adapt across contexts and constraints.

It also reinforced the connection between visual design and implementation. Building the interface made it much clearer how small decisions in spacing, hierarchy, and structure impact the overall experience.
